# Shared Visual Review Protocol

## Scope
This is a visual review of a rendered presentation artifact.

The reviewer sees only:
- PDF
- review brief
- outline

The reviewer does not see:
- code
- source files
- package metadata
- runtime state
- implementation details

## What This Review Is For
This review checks whether the presentation is visually well-made.

It may assess:
- theme
- design
- consistency
- readability
- polish

It must not assess:
- code quality
- technical implementation
- package structure
- runtime correctness
- narrative strength
- content accuracy
- strategic quality of the message

## Composition principle
Symmetry is a tool, not a requirement.
A slide may be symmetrical or asymmetrical.
The question is whether the composition feels intentional, balanced, and well-finished.

## Review method
1. Scan the whole deck once for first impression.
2. Review every slide individually using the slide checklist below.
3. Review the deck as a whole using the deck-level checklist.
4. Write freeform feedback in the reviewer's own voice.

## Slide-by-slide checklist
For every slide, check:

### First-glance read
- Can the viewer understand what kind of slide this is within a few seconds?
- Is there a clear focal point?
- Does the eye know where to go first?

### Hierarchy
- Is the importance structure visually clear?
- Are headline, subhead, body, labels, captions, and evidence clearly separated?

### Readability
- Is the text comfortably readable at presentation scale?
- Is contrast sufficient?
- Does any text feel cramped, tiny, or visually fatiguing?

### Layout
- Does the slide feel balanced?
- If the composition is symmetrical, is that symmetry clean and convincing?
- If the composition is asymmetrical, does it still feel intentional and stable?
- Do paired or opposing regions carry appropriate visual weight?
- Do major blocks feel intentionally placed?
- Does the composition feel stable rather than awkward?

### Alignment
- Are text blocks, media blocks, charts, cards, and edges aligned cleanly?
- Do mirrored, paired, or split elements line up the way the eye expects?
- If the slide suggests symmetry, is that symmetry precise enough to feel deliberate?
- If the slide avoids symmetry, is the structure still coherent and well-resolved?
- Do margins and gutters feel consistent?

### Spacing
- Is there enough breathing room?
- Does spacing help clarify relationships between elements?
- Does spacing feel deliberate?

### Density
- Is the slide overcrowded?
- Is it too empty for its purpose?
- Does the amount of material feel presentation-ready?

### Component quality
- If the slide contains cards, tables, charts, screenshots, callouts, or image blocks, do they look presentation-ready?
- Do they feel integrated into the deck?

### Media treatment
- Are crops, masks, borders, aspect ratios, and framing handled cleanly?
- Are screenshots, charts, and tables visually usable?

### Local polish
- Does anything feel rough, noisy, stretched, cramped, sloppy, or unfinished?

### Slide-to-deck fit
- Does this slide visually belong with the rest of the deck?
- Does it unexpectedly break the visual language?

## Deck-level checklist
After the slide pass, check:

- Does the deck feel like one coherent artifact?
- Do similar slide types behave similarly?
- Is visual rhythm stable across the deck?
- Are there abrupt quality drops between slides?
- Does the deck feel polished end to end?

## Output
The reviewer should produce:
- unstructured freeform feedback
- visual judgment only
- no code suggestions
- no file references
- no implementation talk
