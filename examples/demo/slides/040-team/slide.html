Two-column slide with local slide-owned modules

The demo keeps custom layout additions inside each slide folder instead of a shared deck-level content file.

This lets the deck add custom modules like a hero stat row, a timeline, or a visual placeholder without redefining the global card, badge, or canvas rules.

Slide-local CSS can shape local composition, but colors, typography, shadows, and borders still belong to theme.css.
