The Four Gates

A release is ready when these four checks are all boring

1. Build

  • Correct branch, version, and config are in the candidate build.
  • CI is green and the release notes match what is actually shipping.

2. User path

  • One or two critical flows have been smoke-tested in the target environment.
  • The top known risk has an explicit owner watching it after launch.

3. Operator handoff

  • The release owner, on-call person, and support contact know the ship window.
  • Any dashboards, alert links, or escalation paths are already open.

4. Rollback

  • The team can name the rollback command or switch before pressing go.
  • The stop condition is specific enough that anyone can call it.
