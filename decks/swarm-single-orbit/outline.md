# Swarm Single Orbit Outline

Long decks must lock the story before slide-by-slide buildout.

## Story Arc

- Opening: show the port as the city’s night operating system rather than a passive waterfront.
- Middle: move through the physical, regulatory, labor, utility, safety, and data layers that keep cargo flowing overnight.
- Close: turn the system into a simple operating model leaders can use to spot risk and improve resilience.

## Build Plan

- Total slides: 20
- Batch size: 5
- Build the deck in batches of 5 slides and run `npm run check -- --deck swarm-single-orbit` after each batch.

## Slide Plan

- 01. `010-intro` -> Reframe the port as a night operating system.
- 02. `020-slide-02` -> Explain why night hours matter economically and physically.
- 03. `030-slide-03` -> Map the end-to-end cargo path from berth to inland handoff.
- 04. `040-slide-04` -> Break down the key freight handoffs.
- 05. `050-slide-05` -> Show where customs and compliance enter the flow.
- 06. `060-slide-06` -> Explain the yard as the buffer that absorbs timing mismatch.
- 07. `070-slide-07` -> Explain trucking arrivals, appointments, and gate turns.
- 08. `080-slide-08` -> Explain rail and inland transfer windows.
- 09. `090-slide-09` -> Show the utility layer needed to keep night operations live.
- 10. `100-slide-10` -> Explain labor shifts and role changes across the night.
- 11. `110-slide-11` -> Show the safety command layer.
- 12. `120-slide-12` -> Explain the data flow behind each physical move.
- 13. `130-slide-13` -> Show the control room and dispatch model.
- 14. `140-slide-14` -> Explain overnight maintenance and asset care.
- 15. `150-slide-15` -> Explain security and access control after dark.
- 16. `160-slide-16` -> Surface the most common night-shift bottlenecks.
- 17. `170-slide-17` -> Show how port operations interface with the sleeping city.
- 18. `180-slide-18` -> Show the small set of metrics operators watch every shift.
- 19. `190-slide-19` -> Explain how one failure cascades through the system.
- 20. `200-close` -> Close with a six-part operating model for leaders.
